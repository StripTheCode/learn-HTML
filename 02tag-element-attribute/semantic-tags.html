What is Semantic HTML?

Semantic HTML refers to using meaningful and descriptive HTML tags that clearly define the content’s purpose and structure. Instead of using generic <div> or <span> elements for everything, semantic HTML tags like <header>, <article>, <section>,and <footer> describe the content in a human-readable way.

Why Use Semantic HTML?

  1. Improves Readability: Helps developers and browsers understand the content better.
  2. Better SEO:Search engines can interpret the content structure, improving rankings.
  3. Accessibility:Assistive technologies (like screen readers) can navigate the page easily.
  4. Maintainability:Code is cleaner and easier to manage.

Example of Semantic HTML vs. Non-Semantic HTML

❌ Non-Semantic HTML (Bad Practice)

                
                    <div id="header">My Website </div >
                    <div id="nav">
                        <a href="#">Home</a>
                        <a href="#">About</a>
                        <a href="#">Contact</a>
                    </div>
                        <div id="content">
                        <h1>Welcome!</h1>
                        <p>This is my website.</p>
                    </div >
                    <div id="footer">© 2025 My Website </div>

✅ Semantic HTML (Best Practice)

            
                <header>
                    <h1>My Website</h1>
                </header>
                <nav>
                    <ul>
                        <li><a href="#">Home</a></li>
                        <li><a href="#">About</a></li>
                        <li><a href="#">Contact</a></li>
                    </ul>
                </nav>
                <main>
                    <section>
                        <h1>Welcome!</h1>
                        <p>This is my website.</p>
                    </section>
                </main>
                <footer>© 2025 My Website</footer>

Common Semantic HTML Tags

Tag Purpose
< header > Defines the page header (logo, navigation, etc.).
< nav > Contains navigation links.
< main > Wraps the primary content of the page.
< section > Groups related content.
< article > Represents independent content like blogs/news.
< aside > Sidebar content related to the main content.
< footer > Contains copyright, contact info, etc.
< figure > Used to wrap images, illustrations, or charts.
< figcaption > Provides a caption for <figure>.

Conclusion

Semantic HTML improves SEO, accessibility, and code maintainability. By using the right tags, you make your website more structured, readable, and user-friendly.
